# Инструмент визуализации графа зависимостей  
**Этап 4 — Дополнительные операции**

## Общее описание

Программа представляет собой консольный инструмент для анализа и визуализации графа зависимостей пакетов.  
Она позволяет получать данные о зависимостях из файлов конфигурации, строить граф связей и определять порядок загрузки пакетов.

Приложение разрабатывается пошагово, чтобы постепенно добавить все функции менеджера зависимостей и визуализации графа.  
На данном этапе реализованы дополнительные операции с графом, включая вычисление порядка загрузки зависимостей.

Программа полностью работает через параметры командной строки и поддерживает два режима работы:  
- **test** — используется локальный файл (`Cargo.toml` или тестовый `.txt`),  
- **real** — предназначен для работы с настоящими репозиториями (будет добавлен позднее).

## Основные возможности

- Чтение зависимостей из файла `Cargo.toml` (формат пакетов Rust).  
- Работа с тестовыми графами зависимостей в формате `.txt`.  
- Обработка ошибок и проверка корректности параметров.  
- Построение графа зависимостей и вывод порядка обхода (**BFS**).  
- Определение **порядка загрузки зависимостей** (топологическая сортировка).  
- Поддержка выбора операции через параметр командной строки.  
- Сообщение о циклических зависимостях, если они обнаружены.  

## Настраиваемые параметры

Программа принимает следующие параметры командной строки:

- **--package** — имя анализируемого пакета.  
- **--url** — адрес репозитория (используется для режима `real`).  
- **--path** — путь к локальному репозиторию или тестовому файлу зависимостей.  
- **--mode** — режим работы программы: `real` (реальный) или `test` (тестовый).  
- **--version** — версия анализируемого пакета.  
- **--output** — имя выходного файла, в котором позже будет сохранено изображение графа (например, `graph.svg`).  
- **--operation** — тип выполняемой операции:  
  - `bfs` — обход графа зависимостей (по умолчанию),  
  - `load-order` — вычисление порядка загрузки зависимостей.

## Описание работы

В режиме **test** программа анализирует зависимости локально.  
Если указан путь к директории, ищется файл `Cargo.toml`, из которого извлекаются прямые зависимости пакета.  
Если указан путь к `.txt` файлу, используется тестовый граф зависимостей, где каждая строка задаёт связи между пакетами в виде:
A: B C
B: D
C:
D:

При запуске можно выбрать, какую операцию выполнять:
- `bfs` — обычный обход графа в ширину (BFS);  
- `load-order` — порядок загрузки зависимостей, где зависимости загружаются раньше зависимых пакетов.  

При наличии циклических зависимостей программа выводит предупреждение.

## Примеры запуска программы

Пример 1 — обычный обход графа (BFS):
```bash
python3 main.py --package A --path test_repo.txt --mode test --version 1.0.0 --output graph.svg --operation bfs

python3 main.py --package A --path test_repo_chain.txt --mode test --version 1.0.0 --output graph.svg --operation load-order
