# Инструмент визуализации графа зависимостей  
**Этап 2 — Сбор данных**

## Общее описание

На втором этапе реализована основная логика получения данных о зависимостях пакета.  
Программа теперь может прочитать файл `Cargo.toml` (формат Rust-пакетов) и вывести все прямые зависимости, указанные в нём.  

Пользователь, как и раньше, задаёт параметры через командную строку.  
В режиме `test` программа ищет файл `Cargo.toml` в указанной директории и анализирует его вручную, без использования сторонних библиотек или менеджеров пакетов.  
В дальнейшем полученные данные будут использоваться для построения графа зависимостей.

## Настраиваемые параметры

Программа принимает следующие параметры:

- **--package** — имя анализируемого пакета.  
- **--url** — адрес репозитория (используется для реального режима, пока не реализовано).  
- **--path** — путь к локальному тестовому репозиторию, где находится `Cargo.toml`.  
- **--mode** — режим работы программы: `real` (реальный) или `test` (тестовый).  
- **--version** — версия анализируемого пакета.  
- **--output** — имя выходного файла, в котором позже будет сохранено изображение графа зависимостей (например, `graph.svg`).

Если выбран режим `test`, необходимо указать путь к папке, где лежит `Cargo.toml`.  
Если выбран режим `real`, требуется URL репозитория (на этом этапе не используется).

## Описание работы

Программа открывает файл `Cargo.toml`, находит в нём секцию `[dependencies]` и извлекает все прямые зависимости в формате `имя = версия`.  
Если зависимостей нет, выводится соответствующее сообщение.  
При ошибке (например, если файл не найден) программа сообщает об этом пользователю.
